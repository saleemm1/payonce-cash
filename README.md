## ‚öôÔ∏è Technical Architecture

PayOnce achieves a "Stripe-like" UX by utilizing BCH-specific primitives:

1.  **0-Conf Fulfillment:** We bypass block confirmations (10min+) by monitoring the mempool for broadcasted hashes. This allows Instant Content Unlocking (<2s) while ensuring transaction validity.
2.  **Real-Time Fiat Oracle:** To eliminate volatility risk, we fetch high-frequency price data (`CoinGecko API`) at the millisecond of invoice creation to lock the exact Satoshi value.
3.  **IPFS Encryption:** Files are uploaded to IPFS (InterPlanetary File System) and encrypted. The decryption key is only revealed to the client after the blockchain confirms the payment.
